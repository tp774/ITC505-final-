Process and Creative Choices

The game design started with creating a visually appealing grid layout using CSS. The logic for toggling cells and their neighbors was modularized for better code readability. The board was randomized by simulating random clicks to ensure it was always solveable.

Challenges and Insights

One of the primary challenges was ensuring the board was solveable. Randomly assigning states to cells was not feasible, so simulating random clicks on an all-white board solved this problem. This approach highlighted the importance of planning before coding.

Future Enhancements

- Add a timer or move counter to track player performance.
- Include a reset button to start a new game without refreshing the page.
- Improve accessibility features, such as keyboard controls and ARIA labels.
